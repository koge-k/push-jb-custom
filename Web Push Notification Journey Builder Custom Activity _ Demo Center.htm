Web Push Notification Journey Builder Custom Activity

Clickable messages that come from a website and alert you on your desktop or mobile device even when the website is not open in your browser. All driven from Journey Builder!

Lachlan Ross
October 5th 2017 (Modified on October 17th 2017)

The Web Push Journey Builder Custom Activity allows you to add web browser notifications to your journey interactions. 

The web notifications are clickable messages that come from your website and alert your subscribers on their desktop or mobile device even when the website is not open in their browser!  For more information see Web Push Notification Custom Activity slides below.

Which browsers provide support for web push?

Safari 7.0.3 (With a market share of 5-7%) was the first to provide support for web push. It was followed by Chrome 42 (market share 40-50% and growing) that provided support for both desktop and android. Mozilla Firefox (market share 16-20%) will be soon announcing its support for web push.

Web Push Journey Builder Custom Activity Pre-requisites:

  1. Working MC Learning Account with Journey Builder
  2. Heroku Account - Sign up with your salesforce email address
  3. An account on the MC App Center
  4. An account with Google Firebase (Firebase Cloud Messaging  is the service that delivers the notifications)

Installation Steps:
  1. Create Data Extension to store Web Push Subscriptions/Registrations (See screenshot below)
  2. Connect that Data Extension to contact model in Contact Builder
  3. Create App in MC App Center
  4. Create Google Cloud Messaging Project in Google Firebase
  5. Deploy & Configure Web Push Registration App in Heroku via purple Deploy to Heroku Button
  6. Deploy & Configure Web Push JB Custom Activity in Heroku via purple Deploy to Heroku Button
  7. Register for web push via your newly deployed Web Push Registration App
  8. Build & activate a test Journey that includes a Web Push
